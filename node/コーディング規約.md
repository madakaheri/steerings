# コーディング規約

## 基本規約

[xojs/xo](https://github.com/xojs/xo) のESLintセットを厳守

### xo設定

`package.json` に以下を追加し、`capitalized-comments` だけは無視するよう設定する。

```json
{
	"xo": {
    "rules": {
      "capitalized-comments": "off"
    }
  }
}
```

## Docコメント

JSDoc形式で説明文と入出力を定義

> @see [TypeScript/JSDocリファレンス](https://www.typescriptlang.org/ja/docs/handbook/jsdoc-supported-types.html)

## Lint

xo でリントを行います。

```bash
npm run format
```

## Format

xo でフォーマットを行います。

```bash
npm run format
```

## テスト

node標準のtestモジュールを使用。

```bash
npm test
```

または

```bash
node --test
```

### テストファイルの役割

| ファイル名 | 内容                       |
| ---------- | -------------------------- |
| *.test.js  | 実装時の動作確認（dryrun） |
| *.spec.js  | 仕様を網羅したテスト       |

### テストファイルの配置場所

テスト対象のファイルと同じ場所に同じ名称で配置。

例
```text
├── index.js
├── index.test.js
└── index.spec.js
```
